# Home Panel — Impeccable Design Context

> Generated by the `teach-impeccable` flow. Read by all design skills (`frontend-design`, `polish`, `animate`, `colorize`, `typeset`, `arrange`, `delight`, etc.) before producing any UI code. Edit freely: every change will influence all future design output.

## Design Context

### Users

**Who**: Matteo (owner), housemates and occasional visitors (family, guests).

**Usage context**:
- **Primary panel**: iPad Pro 11" 2nd/3rd gen, wall-mounted or on a shelf in the kitchen/living room with a 3D-printed dock, permanently powered, always landscape, always in foreground (Guided Access), typical reading distance **0.5-3 meters**.
- **Mobile companion**: same Tauri codebase running on Matteo's personal iPhone, used both at home (quick glances, voice commands) and outside (remote control via Tailscale).

**Jobs-to-be-done**:
1. *"I want to see at a glance what's happening at home today"* — weather, family events, waste collection, active timers/alarms, cameras.
2. *"I want to say 'Hey home, add milk to the shopping list' without touching the glass"* — voice-first UX, hands are often busy (cooking, kids, pets).
3. *"I want to feel at home, not in front of an industrial control panel"* — the panel is part of the furniture, not a nuclear power plant dashboard.
4. *"I want to know immediately when someone rings the doorbell and who it is"* — priority interruption that takes over the entire UI.
5. *"I want to trust that only I can access it"* — high perceived security, especially when the alarm/security module arrives.

**Emotions the interface should evoke**: **calm, warmth, trust, quiet control**. Never stress, never overload, never "space command center".

### Brand Personality

**Three words**: **Warm · Crafted · Discreet**

- **Warm**: terracotta/amber palette on warm-tinted neutrals, no steel blue or cyan gradients. A home is a place for living, not a spaceship.
- **Crafted**: every detail earns a smile. Optical typography (Fraunces variable), meditative micro-animations, transitions with natural easing, respects system preferences.
- **Discreet**: the panel lives in the background of home life. It doesn't demand attention when not needed, but when it matters (doorbell, alarm, timer expiring) it commands notice with authority, never with hysteria.

**Voice & Tone (UX writing)**: casual but correct Italian, never childish, never condescending. No "Oops!", no emojis in UI text. Buttons in short imperative form ("Aggiungi", "Spegni", "Apri"). Errors explain what to do next, not what went wrong.

### Aesthetic Direction

**References**:
- **Apple Home / HomeKit**: cleanliness, strong visual hierarchy, tiles that glow with their own light.
- **Daylight Computer**: warm, readable, anti-tech-anxiety.
- **Things 3 (Cultured Code)**: curated typography, meditative animations, generous whitespace.
- **Linear (motion only, NOT colors)**: exemplary easing and transitions (ease-out-quart/expo).
- **A well-kept Italian kitchen**: light wood, terracotta, matte brass, linen. Materials, not gradients.

**Anti-references** (what it must NOT be):
- **Home Assistant default UI / Lovelace**: too engineering-oriented, gauges everywhere, "datacenter" dashboard.
- **Grafana / Kibana**: industrial monitoring panel, cold neutral grays, rows of metrics.
- **Crypto / fintech dashboard**: purple/blue gradients, hero metric with green arrow, decorative sparklines.
- **AI slop 2024-2025**: random glassmorphism, neon on black, purple-to-cyan palette, identical cards with round icon + heading + paragraph, Inter everywhere.
- **Generic Chinese smart-home**: max saturated colors, elastic bouncy animations, flat drop shadows.

**Theme**: **dark + light auto** (follows `prefers-color-scheme` from iOS, manual override in Settings).
- **Light**: warm white `oklch(98% 0.005 80)`, text `oklch(20% 0.02 60)`, surface `oklch(95% 0.01 70)`.
- **Dark**: warm black `oklch(16% 0.012 60)` (never #000), text `oklch(94% 0.008 80)`, surface `oklch(22% 0.014 60)`.
- **Accent**: soft terracotta `oklch(68% 0.15 50)` as primary hue, golden amber `oklch(78% 0.13 75)` as secondary, sage `oklch(72% 0.08 150)` as "calm/success". All warm-tinted in both themes.

**Typography**:
- **Display** (H1-H2, time, large numbers): **Fraunces** variable with `font-optical-sizing: auto` and `font-variation-settings: "SOFT" 50, "WONK" 0`. Round, warm, "softened". Variable from 9 to 144pt.
- **Body** (text, labels, UI): **Geist Sans** variable, refined modern sans-serif without being cold. No Inter, Roboto, Helvetica, or system-ui.
- **Numeric/mono** (times, codes, counts): **Geist Mono** only where semantically appropriate (never as an aesthetic "tech" digit).
- **Scale**: modular fluid `clamp()`, type scale 1.250 (major third) on iPad, 1.200 (minor third) on iPhone.
- **Weights**: 400/500/700 for Geist Sans, 400/600 for Fraunces.

**Layout & space**:
- **iPad landscape primary**: 12-column grid with generous gutter (`clamp(20px, 2.5vw, 32px)`). The home page is a mosaic of differently-sized tiles, not a uniform grid. Never "identical repeated cards".
- **iPhone portrait secondary**: vertical stack, bottom tab navigation, gesture-driven.
- **Spacing scale**: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96 — never identical padding everywhere.
- **Minimum touch target 56pt** (more than Apple's 44pt HIG) because reading/usage distance on the wall-mounted iPad is greater than average and hands are often occupied.
- **Asymmetric**: the home page intentionally breaks the grid for hierarchy (e.g., large weather tile top-left, smaller mosaic to the right).

**Motion**:
- Principle: few animations, each intentional and meditative. Never bouncy, never elastic.
- **Easing**: `cubic-bezier(0.2, 0, 0, 1)` (custom ease-out-quart) for almost everything. `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo) for page transitions.
- **Durations**: 200ms (micro), 320ms (modal, page), 600ms (welcome orchestrations).
- **Only `transform` and `opacity`** — never animate width/height/margin/padding.
- **Page entry**: 40ms stagger between home tiles on first load, none on tab switch.
- **Interrupting states** (doorbell, alarm): the only exception where motion is pronounced (scale + rapid slide + slight device vibration via `Taptic Engine` if available).
- **Reduced motion**: always respected. If `prefers-reduced-motion`, all staggers and slides become pure 120ms fades.

**Visual details**:
- No gratuitous glassmorphism. One blurred surface allowed: the weather/date bar over the camera live view when the doorbell overlay opens.
- No generic gray drop shadows. Warm-tinted shadows with OKLCH `oklch(15% 0.02 60 / 0.18)`.
- Border radius scale: 4 / 8 / 12 / 20 / 32 — 32 only on main home tiles, never on buttons.
- Decorations: thin accent-tinted "fil rouge" lines (1px) as separators instead of visible borders.
- Icons: **Phosphor Icons** weight `duotone` (warm, slightly hand-drawn feel, already used in previous home-panel) or `regular`. Never cold outline, never filled.

### Accessibility & Inclusion

**Primary constraint**: the panel is used by various household members, some of whom may not be familiar with modern apps. It must be "readable by grandma from 2 meters away".

**Requirements**:
- **WCAG 2.2 AA** minimum, AAA for contrast on critical elements (time, today's events, alarm).
- **Minimum touch target 56pt** (vs 44pt HIG), 64pt on primary CTAs.
- **Minimum typography**: 18px body on iPad (vs 16-17 standard), 24px for home labels.
- **AAA contrast** on primary text (`oklch(20% 0.02 60)` on `oklch(98% 0.005 80)` = ~16:1 light, ~14:1 dark).
- **No color as sole channel**: "today" waste status uses icon + label too, not just the bag color.
- **Voice-first**: user should be able to do at least 80% of actions hands-free. Every feature in design phase evaluates "is there a voice command for this?".
- **Reduced motion** always respected.
- **No autoplay audio** except for critical alarms (timer expired, doorbell, home alarm).
- **Visible microphone indicator** always shown when the system is listening for the wake word, for psychological privacy.
- **Native Italian as primary language**, i18n structure ready for English in the future (low priority).

### Design Principles

1. **Quiet by default, loud when it matters** — the panel lives in the background. The only time it takes over is when something truly urgent happens (doorbell, alarm, timer expired). Everything else is discreet.

2. **Voice as a first-class citizen, not a gimmick** — every feature passes the "can I do it hands-free?" test. If the answer is no, it needs rethinking. Voice is not an add-on, it's one of the two primary interaction modes (touch + voice).

3. **Warm before cool** — when choosing between a "tech" solution and a "domestic" one, always pick domestic. Terracotta before blue, Fraunces before Inter, "Aggiungi alla spesa" before "Add item".

4. **Read at three meters** — every piece of home UI must be readable by someone sitting on the couch. If it isn't, it doesn't live on the home page: it lives in a detail view.

5. **Family-aware, not user-aware** — the app has no "users" and "permissions". It has **people**: names, faces, stories. An event belongs to Matteo *and* Cody together. A shopping note was added by whoever spoke, not by `user_id=42`.

6. **Trust over convenience** — when the alarm/security module arrives, every design decision will favor perceived security over speed. Three taps to disarm the alarm is better than one.
